no shutdown
exit

no shutdown

exit

no shutdown

switchport mode access

no shutdown

🔴 Tier 1: Critical (BSOD & Hardware)

Stop errors (Blue Screen of Death) and critical hardware failures. These require immediate triage to determine if repair is possible or if hardware replacement is needed.

Error Name / Context Common Codes Quick Triage Action Escalation Rule
🔴Memory Corruption
0x0000001A
MEMORY_MANAGEMENT
Run memory diagnostics:
mdsched.exe copy
If diagnostics fail or BSOD persists after swapping RAM slots, escalate to Hardware Replacement.
🔴System Process Failure
0x000000EF
CRITICAL_PROCESS_DIED
Run SFC and DISM repair scans in elevated CMD:
sfc /scannow copy
dism /online /cleanup-image /restorehealth copy
If the system cannot boot into Safe Mode to run these scans, escalate to Tier 2 for OS Reimage.
🔴Storage / Boot Drive Failure
0x0000007B
INACCESSIBLE_BOOT_DEVICE
Boot into BIOS/UEFI. Check if SATA/NVMe controller mode changed (AHCI/RAID). Run check disk:
chkdsk c: /f /r copy
If the drive is not detected in BIOS or disk errors cannot be repaired, escalate to Storage Hardware Replacement.
🔴Driver Communication Crash
0x0000009F
DRIVER_POWER_STATE_FAILURE
Identify the crashing driver using driver verifier or check device manager for power saving settings:
verifier copy
If updating the chipset/power drivers doesn't stop the crash, escalate to SysAdmin / Core Image Team.
🔴Kernel Memory Exception
0x00000050
PAGE_FAULT_IN_NONPAGED_AREA
Test RAM and inspect third-party antivirus/virtualization drivers. Isolate newly installed software in Safe Mode.
 If hardware tests pass but the crash loop continues, escalate to Desktop Engineering.
Error Name / Context Common Codes Quick Triage Action Escalation Rule
🟡Device Initialization Failure
Code 10
Device cannot start
Open Device Manager to locate the device, uninstall it, and scan for hardware changes:
devmgmt.msc copy
If a driver rollback or vendor-specific driver update fails to start the device, escalate to Desktop Support for hardware swap.
🟡Driver Corruption / Registry Block
Code 39
Driver load failure
Check for corrupted registry filters or reinstall the driver package. Use pnputil to force driver removal via CMD:
pnputil /delete-driver <oem#.inf> /uninstall copy
If removing registry UpperFilters/LowerFilters or clean reinstalling fails, escalate to SysAdmin / Core Image Team.
🟡Hardware / Component Failure
Code 43
Device stopped by Windows
Power cycle the component. For USB/PCIe issues, disable fast startup or toggle device status via PowerShell:
Disable-PnpDevice -InstanceId "ID" -Confirm:$false copy
Enable-PnpDevice -InstanceId "ID" -Confirm:$false copy
Common on failing GPUs or USB ports. If a full power cycle and re-seat fails, escalate to Hardware Replacement.
🟡Driver Signature Enforcement Block
Code 52
Digital signature invalid
Restart Windows in Advanced Startup mode to temporarily disable driver signature enforcement, or check corporate certificate store. Boot to Settings > System > Recovery > Restart now > Troubleshoot.
 If the driver is mandatory for business operations but unsigned, escalate to InfoSec / Application Packaging Team.
🟡Network Component Incompatibility
0x80071AC3
Network adapter/Wi-Fi dropouts
Reset the entire network stack and flush DNS via elevated prompt:
netsh int ip reset copy
netsh winsock reset copy
ipconfig /flushdns copy
If the network card drops connection under heavy load after a stack reset, escalate to Network Infrastructure Team.
Error Name / Context Common Codes / App Errors Quick Triage Action Escalation Rule
🔵Permission Denied / Blocked
0x80070005
ACCESS_DENIED
Force launch the app with administrative privileges, or clear user profile temp files via CMD:
runas /user:Administrator "C:\Path\To\App.exe" copy
del /q /f /s %temp%\* copy
If access is denied after clearing temp files, escalate to Identity & Access Management (IAM) for NTFS/GPO check.
🔵Microsoft Store App Corruption
0x80246007
0x80072EE7
Reset the Microsoft Store cache and re-register system packages via PowerShell:
wsreset.exe copy
Get-AppXPackage -AllUsers | Foreach {Add-AppxPackage -DisableDevelopmentMode -Register "$($_.InstallLocation)\AppXManifest.xml"} copy
If store-dependent corporate apps fail to download or open, escalate to Application Deployment Team.
🔵Environment / DLL Conflict
0xc000007b
INVALID_IMAGE_FORMAT
Reinstall the application's required runtime dependencies. Repair Microsoft Visual C++ Redistributables or .NET Framework via Programs & Features or direct installer.
 If the 32-bit vs 64-bit architecture mismatch persists after reinstallation, escalate to App Dev / Software Engineering.
🔵Memory Allocation Leak
0xc0000005
STATUS_ACCESS_VIOLATION
Check Task Manager for high background memory usage. Disable third-party browser extensions or hardware acceleration in browser settings.
 If specific corporate web apps crash frequently with this error, escalate to Internal Web Tools Team.
🔵App Initialization Failure
0xc0000142
DLL_INITIALIZATION_FAILED
Check if anti-malware or security agents are blocking the DLL hook. Run a clean boot using configuration tool:
msconfig copy
If the application fails to initialize even in a clean boot environment, escalate to Endpoint Security Team.
Error Name / Context Common Codes / Update Errors Quick Triage Action Escalation Rule
🟢Component Store Corruption
0x800F0988
0x80073701
Reset the Windows Update catalog files, clear the distribution cache, and restart services via CMD:
net stop wuauserv copy
net stop cryptSvc copy
net stop bits copy
ren C:\Windows\SoftwareDistribution SoftwareDistribution.old copy
net start wuauserv copy
If the patch cache reset fails to resolve the loop, escalate to SysAdmin / Systems Management Team (WSUS/SCCM).
🟢Migration / Driver Conflict
0xC1900101
Installation failed in safe_os phase
Disconnect unnecessary external hardware. Run the Windows servicing health check to repair the system image:
DISM.exe /Online /Cleanup-Image /AnalyzeComponentStore copy
DISM.exe /Online /Cleanup-Image /StartComponentCleanup copy
If the feature update fails repeatedly at the exact same percentage, escalate to Desktop Engineering for full OS upgrade log analysis.
🟢Insufficient Disk Space
0x80070070
ERROR_DISK_FULL
Launch the Windows Storage sense utility or run the built-in system file cleanup tool to purge delivery optimization files:
cleanmgr /sageset:1 copy
If system files cannot be purged or user storage exceeds drive capacity limits, escalate to Tier 1 / Hardware Support for a storage upgrade.
🟢Group Policy Block
0x80244007
SOAP client failed
Force a local update of corporate Group Policy objects and verify domain controller connectivity via CMD:
gpupdate /force copy
If the machine fails to sync security policies or reach the local update server, escalate to Active Directory / Network Team.
🟢Service Registration Missing
0x80240438
Update service connectivity
Run the built-in automated maintenance diagnostic engine package via command line utility:
msdt.exe /id WindowsUpdateDiagnostic copy
If network proxy configurations are blocking background update handshakes, escalate to Network Security / Firewall Team.

📄 Event Log IDs Reference

Critical Windows Event Viewer IDs (Application, System, Security logs) mapped by priority tier. Use these to quickly identify root causes without scrolling through thousands of logs.

41
Kernel-Power
System rebooted without cleanly shutting down. Indicates sudden power loss, hard crashes, or pulled power cables.
1001
BugCheck
Computer rebooted from a BSOD. Log contains exact dump file location and stop code parameters for analysis.
7
Disk
Physical bad block detected. Critical warning that hard drive or NVMe SSD is failing and requires immediate replacement.
219
Kernel-PnP
Driver failed to load for a specific device. Usually matches Device Manager Code 10 or Code 43 errors.
10000
WLAN-AutoConfig
EAP (Extensible Authentication Protocol) failure. Wi-Fi card failed to authenticate with corporate network.
1000
Application Error
Faulting application name and module path. Identify exactly which corporate app or DLL crashed.
1002
Application Hang
Program stopped responding and was terminated by user or system. Indicates memory leaks or deadlock loops.
19
WindowsUpdateClient
Installation Failure: Windows failed to install an update with detailed error metadata for troubleshooting.
43
Time-Service
Time provider failed to sync local clock with corporate Active Directory NTP server.

⌨️ PowerShell Snippets

Copy-paste automated PowerShell scripts categorized by urgency. Each snippet solves a common diagnostic or repair task for your tier-based triage workflow.

Get BSOD Bugcheck History
Get-CimInstance -ClassName Win32_NTLogEvent -Filter "Logfile='System' AND EventCode=1001" | Select-Object TimeGenerated, Message | Format-List
Check SMART Status of Storage Drives
Get-PhysicalDisk | Select-Object DeviceId, FriendlyName, OperationalStatus, HealthStatus
List All Failing/Flagged Devices (Device Manager Errors)
Get-CimInstance -ClassName Win32_PnPEntity | Where-Object {$_.ConfigManagerUserConfig -ne 0} | Select-Object Name, Status, DeviceID
Backup All Third-Party System Drivers
Export-WindowsDriver -Online -Destination "C:\DriverBackup"
Find Top 5 Processes Consuming Most Memory (RAM)
Get-Process | Sort-Object WorkingSet -Descending | Select-Object -First 5 Name, Id, @{Name='RAM(MB)';Expression={[math]::Round($_.WorkingSet / 1MB, 2)}}
Force Terminate a Hanging App/Process Instantly
Stop-Process -Name "ProcessName" -Force
Force Run Windows Update Detection and Download Cycle
(New-Object -ComObject Microsoft.Update.AutoUpdate).DetectNow()
Check the Health Status of Component Store (DISM)
Repair-WindowsImage -Online -CheckHealth

👥 Active Directory Troubleshooting

User access, authentication blocks, machine account issues, and replication/group policy syncing. Organized by priority tier for rapid diagnosis and remediation.

🔴 Tier 1: Critical (Domain Authentication & Account Locks)
Context: Employees completely locked out of workstations, or local systems unable to trust the domain.
Common Error / Event ID: Event ID 4625 (Failed logon), "The trust relationship between this workstation and the primary domain failed."
Quick Triage Action:
Reset-ComputerMachinePassword
Escalation Rule: If resetting machine password fails or user remains locked out due to infinite bad-password loop, escalate to IAM / Identity Team.
🟡 Tier 2: Device & Drivers (Computer Object & Joining Issues)
Context: New or re-imaged computers failing to join corporate network or disappearing from directory.
Common Error / Event ID: 0x80072030 (Server not willing to process request), Object Not Found.
Quick Triage Action:
Get-ADComputer -Identity "WorkstationName" -Server "yourdomain.local"
Escalation Rule: If computer object exists but is corrupted or disabled, escalate to Desktop Deployment Team to clear conflicting stale records.
🔵 Tier 3: Apps & Runtime (User Profile & Permission Maps)
Context: Users can log in, but lack proper folder permissions or corporate security groups are missing.
Common Error / Event ID: Event ID 4728 (Member added to security group), Access Denied on shared network drives.
Quick Triage Action:
Get-ADPrincipalGroupMembership -Identity "Username" | Select-Object Name
Escalation Rule: If user belongs to correct group but permissions still denied, escalate to File Server / Storage Administration.
🟢 Tier 4: Updates & Maintenance (SYSVOL & Group Policy Processing)
Context: Background policy changes, logon scripts, or corporate wallpaper updates failing to apply to user devices.
Common Error / Event ID: Event ID 1058 (Cannot access gpt.ini), 0x80070035 (Network path not found).
Quick Triage Action:
Get-WinEvent -LogName "Microsoft-Windows-GroupPolicy/Operational" | Where-Object {$_.LevelDisplayName -eq "Error"} | Select-Object -First 5 TimeCreated, Message
Escalation Rule: If multiple machines fail to pull policies and show SYSVOL errors, escalate immediately to Active Directory Domain Admins (Replication Failure).
